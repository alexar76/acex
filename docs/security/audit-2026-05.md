# ACEX Capital Markets — Security Audit (2026-05)

**Scope:** EVM (`acex/contracts/evm/src/*`) + Solana (`acex/contracts/solana/programs/acex-capital`)  
**Auditor:** Internal pre-release review  
**Status:** Findings addressed in this revision

## Executive summary

ACEX adds listing (ALP), collateral vault, AgentNotes, LiquidityMesh lending, and Pulse AMM on EVM; ALP + collateral on Solana. Design follows AIMarketEscrow patterns: **non-custodial routing**, **pausable admin**, **reentrancy guards**, **auditor gate before listing**.

## Critical

### C1 — Registry ↔ Vault circular deploy left vault.registry unset

**Risk:** `issueAgentNotes` and `lockForNote` callable by anyone if `registry` is zero.

**Fix:** Vault deploys without registry; `setRegistry` one-shot; deploy script orders: Vault → Registry → `setRegistry` → Lending.

---

## High

### H1 — Collateral double-credit on note issuance

**Risk:** `depositCollateral` + `creditCollateral` both called without transfer in early draft.

**Fix:** `issueAgentNotes` transfers USDC from agent to vault, then `creditCollateral` (accounting only).

### H2 — No pause on capital markets

**Fix:** `Pausable` on Registry, Vault, Lending, AMM; Solana `config.paused` flag.

### H3 — Solana vault ATA seeds mismatch

**Risk:** `vault_ata` bump not stored; release uses wrong signer seeds.

**Fix:** Unified seeds `[b"vault", listing_id]` for authority; `[b"vault_ata", listing_id]` for ATA; release uses `vault` authority bump from `ctx.bumps.vault_authority`.

---

## Medium

### M1 — Audit score bypass on `approveListing`

**Fix:** `MIN_AUDIT_SCORE_BPS = 7000`; reject if `UnderAudit` score below threshold.

### M2 — Lending LTV not enforced on collateral drop

**Fix:** `liquidate()` when `collateral * MAX_LTV_BPS < debt * BPS`.

### M3 — AMM no slippage on first pool

**Fix:** `minUsdcOut` / `minShareOut` on swaps; pool creation requires both sides deposited.

### M4 — AgentNotes minted to vault (unredeemable)

**Fix:** Mint to `beneficiary` (agent wallet).

---

## Low

### L1 — `rejectListing` missing delist path

**Mitigation:** Document `delist` as owner follow-up in v0.2.

### L2 — Solana `approve_listing` does not CPI-create mint

**Mitigation:** Admin passes `share_mint` after off-chain SPL mint with PDA authority — documented in deploy runbook.
